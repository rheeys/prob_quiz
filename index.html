Section 01   베이즈 정리 복습

새로운 증거가 나타났다.
확률을 바꿔라.

베이즈 정리는 사전 믿음(prior)을 새로운 증거로 업데이트하는 수학적 방법입니다. "비가 올 것 같다"는 느낌이 검은 구름을 보고 강해지듯, 확률도 새로운 정보를 만나면 바뀌어야 합니다.

P(A | B)  =  P(B | A)  ×  P(A)  /  P(B)
P(A | B)
사후 확률
B를 관찰한 후
A의 확률 (구하는 것)
P(B | A)
우도 (Likelihood)
A가 사실일 때
B가 관찰될 확률
P(A)
사전 확률 (Prior)
증거 B를 보기 전
A에 대한 믿음
P(B)
증거의 총 확률
B가 일어나는
모든 경우의 합
🔑

핵심 직관: 분자는 "A와 B가 함께 일어나는" 경우, 분모는 "B가 일어나는 모든 경우"입니다. P(A|B) = (A∩B 가 B에서 차지하는 비율)

Section 02   예제 1 — 쿠키 항아리

어느 항아리에서
꺼낸 쿠키일까?

문제 설정

상황

항아리 A: 초코 쿠키 30개, 바닐라 쿠키 10개
항아리 B: 초코 쿠키 20개, 바닐라 쿠키 20개

눈을 감고 항아리를 하나 고른 뒤 쿠키를 하나 꺼냈더니 초코 쿠키였습니다.

Q. 항아리 A에서 꺼냈을 확률은?

🍪
항아리 A
초코 30 / 바닐라 10
초코 비율: 75%
🫙
항아리 B
초코 20 / 바닐라 20
초코 비율: 50%
풀이

변수 정의

A = "항아리 A를 선택"  |  B = "초코 쿠키를 꺼냄"

구하는 것: P(A | B) — 초코 쿠키를 꺼냈을 때 항아리 A였을 확률

1
사전 확률 — 눈감고 항아리를 고를 때
P(A) = 1/2 = 0.5  |  P(B항아리) = 1/2 = 0.5
두 항아리를 동등하게 선택
2
우도 — 각 항아리에서 초코가 나올 확률
P(초코 | A) = 30/40 = 0.75
P(초코 | B항아리) = 20/40 = 0.50
3
전체 확률 — 초코 쿠키가 나올 확률
P(초코) = P(초코|A)×P(A) + P(초코|B)×P(B)
= 0.75×0.5 + 0.50×0.5 = 0.625
4
베이즈 정리 적용
P(A | 초코) = P(초코|A) × P(A) / P(초코)
= 0.75 × 0.5 / 0.625 = 0.375 / 0.625
= 0.6 = 60%
사전 확률 50% → 초코를 보고 60%로 업데이트
💡

초코 쿠키라는 증거가 항아리 A의 확률을 50% → 60%로 높였습니다. 항아리 A에 초코가 더 많으니 "A였을 가능성"이 올라간 것입니다. 이것이 베이즈 업데이트의 핵심입니다.

Section 03   예제 2 — 날씨와 우산

구름이 꼈다면
비가 올 확률은?

문제 설정

기상청 통계에 따르면:

조건수치
하루 중 비가 오는 날의 비율P(비) = 20%
비 오는 날에 구름이 낀 비율P(구름|비) = 90%
비 안 오는 날에도 구름이 낀 비율P(구름|맑음) = 40%

Q. 오늘 구름이 꼈습니다. 비가 올 확률은?

나무 다이어그램
하루 100일
☁️ 비 오는 날
20일
P(비) = 0.20
구름+비
18일
×0.90
맑음+비
2일
×0.10
☀️ 맑은 날
80일
P(맑음) = 0.80
구름+맑음
32일
×0.40
맑음만
48일
×0.60
풀이
1
구름이 낀 날의 전체 확률
P(구름) = P(구름|비)×P(비) + P(구름|맑음)×P(맑음)
= 0.90×0.20 + 0.40×0.80
= 0.18 + 0.32 = 0.50
100일 중 구름 낀 날 → 50일 (비 동반 18일 + 맑음 32일)
2
베이즈 정리
P(비 | 구름) = P(구름|비) × P(비) / P(구름)
= 0.90 × 0.20 / 0.50 = 0.18 / 0.50
= 0.36 = 36%
사전 확률 20% → 구름을 보고 36%로 업데이트
💡

구름이 꼈다는 증거가 비 확률을 20% → 36%로 올렸습니다. 하지만 36%에 불과한 이유는, 맑은 날에도 구름이 자주 끼기 때문입니다(40%).
P(비|구름) ≠ P(구름|비) — 구름이 낀 날 비가 올 확률(36%)과, 비 오는 날 구름이 낄 확률(90%)은 완전히 다릅니다.

Section 04   예제 3 — 의료 검사

검사가 양성이면
정말 병에 걸린 걸까?

이 예제는 유병률(사전 확률)의 힘을 가장 극적으로 보여줍니다. 직관과 가장 많이 어긋나는 사례입니다.

문제 설정
조건수치
유병률 (사전 확률)P(병) = 1% (100명 중 1명)
민감도 — 환자가 양성일 확률P(양성|병) = 95%
특이도 — 건강자가 음성일 확률P(음성|건강) = 90%
오탐률P(양성|건강) = 10%

Q. 검사 결과가 양성이라면, 실제로 병에 걸렸을 확률은?

1000명으로 시각화

1000명을 상상해 봅시다.

😷 실제 환자: 10명
→ 양성(올바른 탐지): 9.5명 ≈ 9명
→ 음성(놓침): 0.5명 ≈ 1명
😊 건강한 사람: 990명
→ 음성(올바른 판정): 891명
→ 양성(오탐!): 99명
양성 판정을 받은 사람 총계
진짜 환자(양성): 9명  +  건강한데 양성: 99명  =  총 108명
P(병 | 양성) = 9 / 108 ≈ 8.3%
공식 풀이
1
사전 확률 & 우도
P(병) = 0.01  |  P(건강) = 0.99
P(양성|병) = 0.95  |  P(양성|건강) = 0.10
2
전체 양성 확률
P(양성) = 0.95×0.01 + 0.10×0.99
= 0.0095 + 0.099 = 0.1085
3
베이즈 정리
P(병 | 양성) = 0.95 × 0.01 / 0.1085
≈ 0.0876 ≈ 8.8%
95% 정확도 검사가 양성이어도 실제 병일 확률은 8.8%에 불과!
유병률의 힘

유병률이 달라지면 결과가 극적으로 변합니다.

유병률 양성 시 실제 환자 확률 직관과의 차이
0.1% (1000명 중 1명) 약 1% 극심한 괴리
1% (100명 중 1명) 약 9% 큰 괴리
10% (10명 중 1명) 약 51% 겨우 50%
50% (2명 중 1명) 약 90% 그제서야 직관과 일치
🔑

결론: 검사 정확도보다 유병률(사전 확률)이 훨씬 중요합니다. 이것이 의사들이 증상 없는 사람에게 무분별하게 검사를 권하지 않는 이유입니다.

Section 06   스팸 필터와의 연결

베이즈 정리가
스팸을 잡는 방법

지금까지 배운 베이즈 정리가 스팸 필터에 그대로 적용됩니다. 변수만 바뀔 뿐입니다.

대응 관계
의료 검사 스팸 필터
병에 걸렸다 / 건강하다 스팸이다 / 정상이다
검사 결과 (양성/음성) 특정 단어가 등장
유병률 P(병) 사전 확률 P(스팸)
민감도 P(양성|병) P(단어 | 스팸)
오탐률 P(양성|건강) P(단어 | 정상)
P(병 | 양성) P(스팸 | 단어) ← 구하는 것
스팸 필터 공식
P(스팸 | "클릭") = P("클릭" | 스팸) × P(스팸) / P("클릭")
P("클릭" | 스팸) = "클릭"이 스팸 이메일에 등장하는 확률 → 학습 데이터로 계산
P(스팸) = 스팸 이메일의 사전 비율
P("클릭") = "클릭"이 등장하는 전체 확률 (정규화 상수)
구체적 계산 예시

학습 데이터: 스팸 5개, 정상 5개 / 선택 키워드 5개 (V=5)

"클릭"이 스팸 3개에 등장, 정상 0개에 등장 (라플라스 스무딩 적용)

1
우도 계산 (라플라스 스무딩 포함)
P("클릭"|스팸) = (3+1)/(5+5) = 0.40
P("클릭"|정상) = (0+1)/(5+5) = 0.10
2
사전 확률 (균등 가정)
P(스팸) = 0.5  |  P(정상) = 0.5
3
베이즈 정리 적용
P(스팸|"클릭") = 0.40×0.5 / (0.40×0.5 + 0.10×0.5)
= 0.20 / (0.20 + 0.05) = 0.20 / 0.25
= 0.80 = 80%
"클릭"이 있는 이메일은 스팸 확률 80%
↓

이제 실습으로!

단어 하나가 아닌 여러 단어를 동시에 사용하면 — 나이브 베이즈.
각 단어의 확률을 모두 곱합니다.

P(스팸 | 클릭, 무료, 이벤트) ∝ P(스팸) × P(클릭|스팸) × P(무료|스팸) × P(이벤트|스팸)

Section 07   나이브 베이즈

단어가 여러 개일 때 —
확률을 곱한다

베이즈 정리는 단어 하나에 대한 확률만 계산합니다. 하지만 이메일에는 단어가 수십 개 있습니다. 나이브 베이즈는 여러 단어를 동시에 증거로 사용하는 방법입니다.

핵심 가정 — "나이브"한 이유

조건부 독립 가정

각 단어가 서로 독립이라고 가정합니다. 즉, "클릭"이 등장했다는 사실이 "무료"의 등장 확률에 영향을 주지 않는다고 봅니다.

🤔

현실에서 이 가정은 틀립니다. 스팸 메일에서 "클릭"과 "무료"는 함께 나오는 경향이 있습니다. 그런데도 나이브 베이즈는 실용적으로 잘 작동합니다 — 이것이 이 알고리즘의 역설적 매력입니다.

공식 확장

단어가 하나일 때:

P(스팸 | 클릭) ∝ P(스팸) × P(클릭 | 스팸)

단어가 여러 개일 때 — 우도를 모두 곱합니다:

P(스팸 | 클릭, 무료, 이벤트) ∝
  P(스팸) × P(클릭 | 스팸) × P(무료 | 스팸) × P(이벤트 | 스팸)
∝ 기호는 "비례한다"는 뜻입니다. 분모 P(B)는 정상·스팸 양쪽에 동일하게 적용되므로 비교할 때는 생략하고 큰 쪽을 스팸으로 판정합니다.
구체적 계산 예시

학습 데이터: 스팸 5개, 정상 5개 / 키워드 5개 선택 (V=5)

단어 스팸 등장 정상 등장 P(단어|스팸) P(단어|정상)
클릭 3회 0회 (3+1)/(5+5) = 0.400 (0+1)/(5+5) = 0.100
무료 2회 0회 (2+1)/(5+5) = 0.300 (0+1)/(5+5) = 0.100
안내 0회 4회 (0+1)/(5+5) = 0.100 (4+1)/(5+5) = 0.500

테스트 이메일: "무료 강습 신청 안내"

포함된 키워드: 무료, 안내

1
스팸 점수 계산
S = P(스팸) × P(무료|스팸) × P(안내|스팸)
= 0.5 × 0.300 × 0.100 = 0.0150
2
정상 점수 계산
H = P(정상) × P(무료|정상) × P(안내|정상)
= 0.5 × 0.100 × 0.500 = 0.0250
3
사후 확률 (정규화)
P(스팸 | 이메일) = S / (S + H) = 0.0150 / 0.0400
= 37.5% → 정상 판정
"무료"는 스팸 신호지만 "안내"라는 강한 정상 신호가 이를 상쇄했습니다.
단일 vs 복합 증거 비교
방법 사용 증거 특징
단순 베이즈 단어 1개 (가장 변별력 높은 것) 빠르지만 정보 손실 큼
나이브 베이즈 선택한 키워드 전체 더 많은 정보 활용, 일반적으로 더 정확
💡

"무료"만 보면 스팸 점수가 높지만, "안내"를 함께 보면 정상으로 바뀝니다. 여러 증거를 함께 쓸수록 판단이 정교해집니다.

Section 08   라플라스 스무딩

확률이 0이 되면
모든 게 0이 된다

나이브 베이즈의 핵심 문제: 학습 데이터에 한 번도 등장하지 않은 단어가 있으면 어떻게 될까요?

문제 상황

학습 데이터에 없는 단어

"클릭"이 정상 이메일에서 한 번도 등장하지 않았다고 가정합니다.

!
스무딩 없는 경우
P(클릭 | 정상) = 0 / 5 = 0.000
이제 정상 점수 계산: H = P(정상) × ... × P(클릭|정상) × ...
= 0.5 × ... × 0 × ... = 0
💥

단 한 단어의 확률이 0이 되면, 나머지 모든 단어의 확률을 곱해도 결과가 0입니다. 다른 단어들이 아무리 정상 신호를 보내도, "클릭"이 정상 데이터에 없었다는 이유만으로 이 이메일은 무조건 스팸으로 판정됩니다.

해결책 — 라플라스 스무딩

아이디어: 가상의 관찰 1회 추가

모든 단어가 각 클래스에서 최소 1번은 관찰됐다고 가정합니다.

P(단어 | 클래스) = (등장 횟수 + 1) / (클래스 이메일 수 + V)
+1 → 분자: 가상 관찰 1회 추가
+V → 분모: V개 단어에 각 1회씩 추가했으므로 총 V 증가
V = 선택한 키워드의 총 개수
전
스무딩 없음 (V=5, 정상 5개)
P(클릭 | 정상) = 0 / 5 = 0.000
후
라플라스 스무딩 적용 (V=5, 정상 5개)
P(클릭 | 정상) = (0+1) / (5+5) = 0.100
0이 아닌 작은 값 → 곱셈에서 다른 단어들의 정보가 살아남음
왜 분모에 V를 더하나?

분자에 +1을 V번(단어 수만큼) 했으므로, 분모에도 그 합인 +V를 더해야 전체 확률의 합이 1로 유지됩니다.

V=3 키워드, 정상 이메일 5개일 때 확률의 합:
P(클릭|정상) + P(무료|정상) + P(안내|정상)
= (0+1)/(5+3) + (0+1)/(5+3) + (4+1)/(5+3)
= 1/8 + 1/8 + 5/8 = 7/8 ≠ 1
* 베르누이 나이브 베이즈에서는 단어의 등장/미등장이 독립적이므로
합이 반드시 1일 필요는 없습니다. 분모 +V는 확률 추정의 편향(bias)을
균등하게 보정하기 위한 장치입니다.
